# Report provenance standard

## Purpose

Project-control reports must record provenance by source type and authority, not by external AI model or vendor branding.

This standard applies to project-control reports and report-like planning or audit documents. It helps reviewers distinguish repository evidence, advisory source material, and human approval without giving external systems implied authority.

## Rule

- Internal repository automation or project-control mechanisms may be described by role when needed.
- External model/vendor names should not be recorded in reports unless required for reproducibility or legal/audit reasons.
- External inputs should be described generically, for example:
  - maintainer-provided external advisory
  - third-party advisory source
  - non-repository advisory input
  - external model-assisted summary
  - maintainer-provided source summary

## Recommended provenance block

```text
Provenance:

- Requested by: human maintainer
- Drafted by: controlled assistant or repository automation
- Reviewed by: project review, PR review, and CI
- External input: none / maintainer-provided external advisory / third-party advisory source
- External model/vendor: not recorded unless required for reproducibility
- Evidence status: advisory, non-canonical, not independently verified unless source artifact is stored in repository
- Authority: advisory only
- Final approval: human maintainer
```

## Wording examples

Bad:

- Generated by [external model/vendor name]
- Based on [external model/vendor name] analysis
- [external model/vendor name] said/recommended

Good:

- Based on a maintainer-provided external advisory
- Based on a non-repository advisory source
- Source type: external model-assisted summary, vendor not recorded
- Evidence status: advisory and non-canonical

## Authority boundary

Reports do not gain authority from the model/tool that drafted or suggested them.

Authority comes from repository review, CI, audit trail, and human maintainer approval.

## Exception rule

If a model/tool name is part of a canonical witness record, QR record, hash record, agent identity file, or historical record identity, do not normalize it through this report standard. Those records are outside this report-provenance rule.

## HC boundary markers

```text
advisory_only=true
public_safe=true
truth_guarantee=false
human_review_required=true
repository_mutation=false except documentation provenance wording
approval_authority=false
merge_authority=false
```
