# Security Policy Finding Status

Status: advisory checkpoint.

The outside review raised concern that sensitive vulnerability details should not be directed to public GitHub Issues.

Current `SECURITY.md` already separates public integrity reports from private vulnerability reports.

- public GitHub Issues are limited to non-sensitive public integrity concerns;
- sensitive vulnerability reports must not be posted publicly;
- private vulnerability reporting is preferred when available;
- public requests must not include exploit details, private records, secrets, credentials, active abuse details, or uncertain disclosure material.

This means the reported concern does not match the current security policy text.
