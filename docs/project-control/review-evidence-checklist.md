# Review Evidence Checklist

Status: repository operating note.

Before a pull request is accepted, reviewers should confirm repository evidence directly:

1. current PR number and latest head SHA;
2. changed files and diff match the task;
3. protected-path impact is understood;
4. checks completed successfully on the latest head SHA;
5. PR comments and review threads are checked;
6. unresolved Codex or reviewer findings are resolved;
7. duplicate PRs for the same task are handled first;
8. post-acceptance state is verified from GitHub.

This note applies to humans, assistants, bots, and delegated reviewers.

Repository evidence outranks chat memory and screenshots.

Human final authority remains required. `truth_guarantee=false` remains the default boundary.
