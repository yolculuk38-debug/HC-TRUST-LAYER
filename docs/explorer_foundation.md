# Explorer Foundation (Advisory Layer)

## Purpose

The HC:// explorer foundation is a visibility layer for HC-TRUST-LAYER operators and reviewers. It improves discoverability of canonical records without changing canonical record semantics, verification policy, or trust-kernel behavior.

## Scope

The lightweight explorer index is generated by `src/generate_explorer_index.py` and currently scans only canonical record directories:

- `records/pending/`
- `records/verified/`
- `records/archived/`

It outputs `generated/explorer_index.json` for read-only navigation and review support.

## Boundary Rules

- The explorer is **not** a truth engine.
- The explorer does **not** replace human-supervised validation.
- The explorer indexes canonical records only.
- Generated explorer files are non-canonical artifacts and are excluded from strict canonical record validation.
- Canonical schema, validator logic, and trust-kernel guarantees remain unchanged.

## Current Index Fields

Each indexed entry includes:

- `record_id`
- `verification_status`
- `timestamp`
- `author`
- `content_hash_prefix`
- `source_file`

Entries are deterministically sorted by `timestamp`, then `record_id`.

## CI Integration

In CI, explorer index generation runs only after canonical validation and hash checks succeed. The generated file is uploaded as an artifact for audit and review workflows.

## Forward Preparation

This foundation prepares future HC:// work such as:

- QR-linked verification views
- public viewer UX
- federation-aware visibility surfaces
- broader trust-layer operator tooling

All future expansion remains advisory-first and subject to human-supervised validation.
