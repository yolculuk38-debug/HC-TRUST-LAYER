# HC-TRUST-LAYER HC:// Provenance Viewer Foundation

## Status

- documentation-only foundation
- no runtime viewer implementation in this phase
- no production readiness claim

## Provenance Viewer Overview

The **provenance viewer** is a planned inspection surface for provenance and audit trail continuity in HC-TRUST-LAYER.

It is intended to help public and reviewer audiences inspect evidence lineage without changing canonical verification behavior.

## Provenance Chain Concepts

Provenance chain concepts include:

- origin references
- transformation and revision events
- supersession lineage
- continuity checks across linked events

The provenance viewer should present chain steps in stable chronological and relationship-aware form.

## Witness Visibility

Witness visibility should include:

- witness identity class (human/AI/system)
- witness timestamp context
- witness-to-canonical-record linkage
- witness contribution scope

Witness visibility should remain transparent and support human-supervised validation interpretation.

## Validator Linkage Visibility

Validator linkage visibility should include:

- validator references attached to provenance events
- outcome categories per validation event
- validator evidence pointers where available
- cross-reference to trust graph relationships

## Revision History Visibility

Revision history visibility should show:

- prior and superseding canonical record relations
- revision rationale metadata where available
- continuity or branch divergence indicators
- associated audit trail checkpoints

## Verification Package Relationship

The provenance viewer should show how provenance chain segments map to a verification package:

- package-to-event coverage boundaries
- missing-event indicators when package scope is partial
- package version context for reproducibility interpretation

## Snapshot Relationship

The provenance viewer should show provenance relationship to verification snapshots:

- snapshot capture points
- snapshot coverage intervals
- changes since snapshot indicators
- reproducibility notes for replayed verification results

## Replay/Dispute Indicators

Replay/dispute indicators should include:

- replay cluster markers
- disputed lineage edge markers
- unresolved dispute state visibility
- escalation pointers for human-supervised validation

## Audit Trail Visualization Concepts

Audit trail visualization concepts include:

- append-oriented event timeline
- event type legends (verify/revise/revoke/dispute)
- actor and validator linkage overlays
- continuity gap highlighting

## Provenance Continuity Concepts

Provenance continuity concepts include:

- deterministic linkage across chain events
- orphan event detection visibility
- contradiction visibility for competing lineage claims
- explicit unknown/insufficient-evidence states

## Terminology Alignment

This foundation preserves HC-TRUST-LAYER and HC:// terminology, including provenance, audit trail, verification package, canonical record, trust graph, and human-supervised validation.
