# HC:// Secure Analysis Guidelines

## Purpose
This document defines safe analysis and compatibility review principles.

## Allowed Areas
- interoperability analysis
- protocol compatibility review
- audit inspection
- defensive verification research
- integrity validation

## Restricted Areas
- unauthorized intrusion
- credential theft
- malware delivery
- destructive operations
- bypass instructions

## Core Principle
All analysis activity must prioritize safety, legality, transparency, and defensive verification.
