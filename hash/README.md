# Hash Verification Layer

This directory contains hash-related references for Humanity Chain archive records.

## Purpose

The hash layer is designed to support basic integrity verification for archived interaction records.

A hash reference may be used to show that a record, snapshot, or archive file has not been silently changed after documentation.

## Supported Hash Type

Current preferred format:

- SHA-256

## Record Hash Format

Each hash record may include:

- Record ID
- Date
- File path
- SHA-256 hash reference
- Archive status
- Optional QR-linked verification path

## Important Notice

Hash references support transparency and integrity checking.

They do not prove that the original content is true.  
They only help verify whether a documented record has changed after being archived.

Human review, source context, public revision history, and transparent documentation remain part of the verification process.
